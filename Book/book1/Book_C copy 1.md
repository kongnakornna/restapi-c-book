## 1.2 ภาพรวมของ C# และ .NET ในปี 2026

### 1.2.1 C# คืออะไร?

**C#** (อ่านว่า “ซีชาร์ป”) เป็นภาษาโปรแกรมเชิงวัตถุที่พัฒนาโดย Microsoft ภายใต้การนำของ Anders Hejlsberg (ผู้ออกแบบ Turbo Pascal และ Delphi) ภาษา C# ถูกเปิดตัวครั้งแรกในปี 2000 พร้อมกับ .NET Framework ปัจจุบัน (ปี 2026) C# อยู่ในเวอร์ชัน 14 (หรือตามรอบการปล่อยของ .NET) ซึ่งมาพร้อมกับฟีเจอร์ใหม่ ๆ เช่น primary constructors, collection expressions, pattern matching ที่ทรงพลัง, และการทำงานกับ Span<T> ที่มีประสิทธิภาพสูง

**ลักษณะเด่นของภาษา C#:**

- **Strongly typed (ชนิดข้อมูลแข็งแกร่ง):** ตัวแปรทุกตัวต้องมีชนิดข้อมูลที่แน่นอน (เช่น int, string, bool) ข้อผิดพลาดเกี่ยวกับชนิดข้อมูลจะถูกตรวจพบตั้งแต่ตอนคอมไพล์ ทำให้โค้ดปลอดภัยและบำรุงรักษาง่าย
- **Object-Oriented (เชิงวัตถุ):** รองรับการสืบทอด (inheritance), การห่อหุ้ม (encapsulation), และพหุสัณฐาน (polymorphism) อย่างสมบูรณ์
- **Garbage collected:** มีตัวเก็บขยะ (Garbage Collector) ที่จัดการหน่วยความจำอัตโนมัติ คุณไม่ต้องเรียก free หรือ delete เอง
- **Language Integrated Query (LINQ):** ความสามารถในการเขียนคำสั่ง查询คล้าย SQL ภายในภาษา C# เพื่อทำงานกับคอลเลกชันและฐานข้อมูล
- **Asynchronous programming:** รองรับคีย์เวิร์ด `async` และ `await` ทำให้เขียนโปรแกรมที่ไม่บล็อก (non‑blocking) ได้ง่าย
- **Cross-platform:** โค้ด C# ที่เขียนสามารถรันได้บน Windows, Linux, macOS, iOS, Android, และแม้แต่เว็บเบราว์เซอร์ผ่าน WebAssembly

### 1.2.2 .NET คืออะไร?

**.NET** (อ่านว่า “ดอทเน็ต”) คือแพลตฟอร์มสำหรับพัฒนาและรันแอปพลิเคชัน ประกอบด้วย:

1. **.NET Runtime (Common Language Runtime – CLR)** – เครื่องเสมือนที่จัดการการทำงานของโค้ดที่คอมไพล์แล้ว: โหลดโค้ด, จัดสรรหน่วยความจำ, จัดการเธรด, ตรวจสอบความปลอดภัย, และเก็บขยะ
2. **.NET Class Library** – ชุดไลบรารีมาตรฐานขนาดใหญ่ที่มีคลาสสำเร็จรูปสำหรับงานทั่วไป เช่น การอ่านเขียนไฟล์, การเชื่อมต่อเครือข่าย, การเข้ารหัส, การทำงานกับ JSON/XML, การเชื่อมต่อฐานข้อมูล, และอื่น ๆ
3. **SDK (Software Development Kit)** – เครื่องมือสำหรับนักพัฒนา: คอมไพลเลอร์ (C# compiler), dotnet command-line tool, เทมเพลตโปรเจกต์, และเอกสาร API

**วิวัฒนาการของ .NET แบบย่อ:**

| เวอร์ชัน | ปีที่เปิดตัว | จุดเด่น |
|----------|--------------|---------|
| .NET Framework 1.0–4.8 | 2002–2019 | ทำงานบน Windows เท่านั้น, ติดตั้งแยกจากแอป |
| .NET Core 1.0–3.1 | 2016–2019 | ข้ามระบบ (Windows, Linux, macOS), โอเพนซอร์ส, เล็กและเร็ว |
| .NET 5, 6, 7, 8 | 2020–2024 | รวม .NET Framework และ .NET Core เข้าด้วยกัน, LTS ทุก 2 ปี |
| .NET 9, 10 (2026) | 2025–2026 | ปรับปรุงประสิทธิภาพ AOT (Native AOT), เวอร์ชันปัจจุบัน |

ในหนังสือเล่มนี้เราจะใช้ **.NET 9 หรือ .NET 10** (ขึ้นอยู่กับช่วงเวลาที่คุณอ่าน) ซึ่งเป็นเวอร์ชันที่มีการสนับสนุนระยะยาว (LTS – Long Term Support) หรือ STS (Standard Term Support) ก็ได้ โดยฟีเจอร์ทั้งหมดที่สอนจะเข้ากันได้กับ .NET 6 ขึ้นไป

### 1.2.3 ความสัมพันธ์ระหว่าง C# และ .NET

**C# เป็นภาษา, .NET เป็นแพลตฟอร์ม** คุณไม่สามารถรันโค้ด C# ได้โดยไม่มี .NET เช่นเดียวกับที่คุณไม่สามารถรัน Java ได้โดยไม่มี JVM (Java Virtual Machine) กระบวนการทำงาน:

1. คุณเขียนโค้ด C# (ไฟล์ `.cs`)
2. คอมไพลเลอร์ C# (csc.exe หรือ dotnet build) แปลงโค้ดของคุณเป็นภาษา Intermediate Language (IL) – คล้ายกับ bytecode ของ Java
3. เมื่อรันโปรแกรม, CLR จะแปล IL เป็น machine code (ผ่าน JIT – Just‑In‑Time compiler) หรือถ้าใช้ Native AOT จะแปลล่วงหน้า
4. CLR จะจัดการหน่วยความจำ, เธรด, และเรียกใช้ไลบรารีพื้นฐาน

### 1.2.4 ประเภทของแอปพลิเคชันที่สร้างด้วย C# และ .NET

คุณสามารถสร้างแอปพลิเคชันได้หลากหลายประเภท:

- **เว็บแอปและ REST API** – ด้วย ASP.NET Core (Minimal API, MVC, Razor Pages, Blazor)
- **เดสก์ท็อป** – Windows Forms (WinForms), WPF (Windows Presentation Foundation), .NET MAUI (ข้ามแพลตฟอร์ม)
- **มือถือ** – .NET MAUI, Xamarin (กำลังถูกแทนที่ด้วย MAUI)
- **เกม** – Unity Game Engine (ใช้ C# เป็นภาษาหลัก)
- **คลาวด์และไมโครเซอร์วิส** – Azure Functions, AWS Lambda, Google Cloud Functions
- **อุปกรณ์ IoT** – .NET IoT Libraries รองรับ Raspberry Pi, Arduino
- **AI และ Machine Learning** – ML.NET, TensorFlow.NET

หนังสือเล่มนี้จะเน้นที่การพัฒนา **REST API แบบ Backend** เป็นหลัก แต่จะสอดแทรกตัวอย่าง WPF และ Unity ในภาค 4 เพื่อให้เห็นความหลากหลาย

## 1.5 สัญลักษณ์และรูปแบบที่ใช้ในหนังสือ

เพื่อให้การอ่านเป็นไปอย่างราบรื่น หนังสือใช้สัญลักษณ์และรูปแบบมาตรฐานดังนี้:

| สัญลักษณ์ | ชื่อ | ความหมาย |
|-----------|------|-----------|
| 💡 | เคล็ดลับ (Tip) | ข้อแนะนำหรือเทคนิคที่ช่วยให้เข้าใจหรือเขียนโค้ดได้ดีขึ้น มักเป็นสิ่งที่ผู้เขียนค้นพบจากประสบการณ์จริง |
| ⚠️ | ข้อควรระวัง (Warning) | จุดที่มักเกิดข้อผิดพลาดบ่อย, กับดักทางภาษา, หรือสิ่งที่ควรระวังเป็นพิเศษ |
| 📝 | หมายเหตุ (Note) | ข้อมูลเพิ่มเติมที่ไม่จำเป็นต่อความเข้าใจหลัก แต่มีประโยชน์สำหรับผู้อ่านที่อยากรู้ลึก |
| 🧪 | แบบฝึกหัด (Exercise) | โจทย์ให้คุณลองทำด้วยตนเอง ก่อนดูเฉลย |
| 🔗 | แหล่งอ้างอิง (Reference) | ลิงก์ไปยังเอกสารภายนอก (Microsoft Docs, บทความ, วิดีโอ) |
| 📄 | ตัวอย่างโค้ด (Code Example) | บล็อกโค้ดที่สามารถคัดลอกไปรันได้ทันที |
| ✅ | จุดตรวจสอบ (Checkpoint) | คำถามทบทวนสั้น ๆ ก่อนไปหัวข้อถัดไป |

**มาตรฐานการเขียนโค้ดในหนังสือ:**

- **การตั้งชื่อ:**
  - ตัวแปรท้องถิ่นและพารามิเตอร์: `camelCase` (เช่น `studentName`, `totalPrice`)
  - คลาส, เมธอด, พร็อพเพอร์ตี้, ค่าคงที่ (public): `PascalCase` (เช่น `CalculateTotal`, `StudentManager`)
  - ฟิลด์ private: `_camelCase` (underscore นำหน้า) หรือ `camelCase` ขึ้นอยู่กับสไตล์团队的
  - ค่าคงที่ (constant): `PascalCase` หรือ `UPPER_CASE` แล้วแต่บริบท

- **วงเล็บปีกกา (braces):** เราใช้สไตล์ **Allman** (วงเล็บปีกกาขึ้นบรรทัดใหม่) สำหรับเมธอดและคลาสขนาดใหญ่ แต่ในตัวอย่างสั้น ๆ อาจใช้ **K&R** เพื่อประหยัดพื้นที่

```csharp
// Allman style (มาตรฐานของหนังสือ)
public void MyMethod()
{
    Console.WriteLine("Hello");
}

// K&R style (ใช้ในตัวอย่างสั้น ๆ เพื่อประหยัดพื้นที่)
public void MyMethod() {
    Console.WriteLine("Hello");
}
```

- **การจัดวาง (indentation):** ใช้ 4 ช่องว่าง (ไม่ใช้แท็บ)

## 1.6 ตัวอย่างโค้ดแรกของคุณ (Hello World สองรูปแบบ)

ก่อนที่เราจะลงรายละเอียดในบทถัดไป เราจะให้คุณลองเขียนโปรแกรมแรกใน C# เพื่อสร้างความคุ้นเคย แม้คุณอาจยังไม่เข้าใจทุกคำสั่ง – ไม่เป็นไร เพียงแค่รันให้ได้ก็ถือว่าประสบความสำเร็จ

### ตัวอย่างที่ 1.1: Hello World แบบดั้งเดิม (มีคลาสและเมธอด Main)

```csharp
// ตัวอย่างที่ 1.1: HelloWorldTraditional.cs
using System;

namespace MasteringCSharp.Basics
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("========================================");
            Console.WriteLine("  ยินดีต้อนรับสู่ Mastering C# .NET 2026");
            Console.WriteLine("========================================");
            Console.Write("กรุณาพิมพ์ชื่อของคุณ: ");
            string name = Console.ReadLine();
            Console.WriteLine($"สวัสดี {name} ขอให้สนุกกับการเรียนรู้ C# นะครับ/คะ");
            Console.WriteLine("\nกด Enter เพื่อปิดโปรแกรม...");
            Console.ReadLine();
        }
    }
}
```

**วิธีการรัน:**
1. ติดตั้ง .NET SDK (ดูบทที่ 7)
2. เปิด Terminal หรือ Command Prompt
3. สร้างโฟลเดอร์ใหม่ `mkdir MyFirstApp` แล้ว `cd MyFirstApp`
4. พิมพ์ `dotnet new console` – จะได้ไฟล์ `Program.cs` มา
5. แทนที่เนื้อหาใน `Program.cs` ด้วยโค้ดด้านบน
6. พิมพ์ `dotnet run` – โปรแกรมจะทำงาน

**ผลลัพธ์ที่คาดหวัง (ตัวอย่าง):**
```
========================================
  ยินดีต้อนรับสู่ Mastering C# .NET 2026
========================================
กรุณาพิมพ์ชื่อของคุณ: สมชาย
สวัสดี สมชาย ขอให้สนุกกับการเรียนรู้ C# นะครับ/คะ

กด Enter เพื่อปิดโปรแกรม...
```

### ตัวอย่างที่ 1.2: Hello World แบบ Top-level statements (C# 9 ขึ้นไป – แนะนำ)

ตั้งแต่ C# 9 (มาพร้อม .NET 5) คุณสามารถเขียนโปรแกรมโดยไม่ต้องมีคลาสและเมธอด `Main` อย่างชัดเจน เรียกว่า **Top-level statements** วิธีนี้ช่วยลดความยุ่งยากสำหรับโปรแกรมสั้น ๆ และบทเรียนแรก ๆ

```csharp
// ตัวอย่างที่ 1.2: HelloWorldTopLevel.cs
// ไม่มี using? มีได้! using ยังคงต้องเขียน

using System;

Console.WriteLine("╔══════════════════════════════════════╗");
Console.WriteLine("║   Mastering C# .NET 2026 - Hello!   ║");
Console.WriteLine("╚══════════════════════════════════════╝");
Console.Write("What's your name? ");
string? userName = Console.ReadLine();
Console.WriteLine($"Hello, {userName}! Welcome to the journey.");
Console.WriteLine("Press any key to exit...");
Console.ReadKey();
```

> 💡 **เคล็ดลับ:** ตลอดหนังสือเล่มนี้ เราจะใช้ Top-level statements ในตัวอย่างสั้น ๆ เป็นหลัก เพราะช่วยให้คุณโฟกัสที่แนวคิดหลัก ไม่ใช่ boilerplate code

### การลองผิดลองถูก (Experiment)

หลังจากรันตัวอย่างได้แล้ว ให้ลองแก้ไขเล็กน้อย:
- เปลี่ยนข้อความใน `Console.WriteLine`
- เพิ่มบรรทัด `Console.ForegroundColor = ConsoleColor.Green;` ก่อนพิมพ์ข้อความ
- ลองใช้ `Console.Beep()` เพื่อให้เครื่องส่งเสียงบี๊บ

การทดลองด้วยตนเองคือวิธีที่ดีที่สุดในการเรียนรู้

## 1.8 แบบฝึกหัดท้ายบท (3–5 ข้อ)

🧪 **แบบฝึกหัดที่ 1 (ความรู้ทั่วไป):**  
อธิบายความแตกต่างหลักระหว่าง .NET Framework (แบบเก่า) กับ .NET 5/6/8/9/10 ในปัจจุบัน ตอบอย่างสั้น 2–3 ประโยค โดยเน้นเรื่องความสามารถในการข้ามระบบปฏิบัติการ (cross-platform)

🧪 **แบบฝึกหัดที่ 2 (การเขียนโค้ด):**  
ให้เขียนโปรแกรม C# โดยใช้ Top-level statements ที่รับชื่อผู้ใช้และอายุ (เป็นจำนวนเต็ม) ทางคอนโซล แล้วแสดงข้อความ “สวัสดี [ชื่อ], คุณอายุ [อายุ] ปี” หลังจากนั้นให้แสดงข้อความ “ในอีก 10 ปีข้างหน้า คุณจะอายุ [อายุ+10] ปี” (ใช้ string interpolation)

🧪 **แบบฝึกหัดที่ 3 (ความเข้าใจโครงสร้างหนังสือ):**  
หนังสือเล่มนี้มีทั้งหมดกี่ภาค แต่ละภาคชื่ออะไรบ้าง และภาคใดมีจำนวนบทมากที่สุด (ดูจากสารบัญในหัวข้อ 1.3)

🧪 **แบบฝึกหัดที่ 4 (การวิเคราะห์):**  
จงบอกประโยชน์หลักของ `Console.WriteLine()` และ `Console.ReadLine()` พร้อมยกตัวอย่างสถานการณ์ในชีวิตจริง (ที่ไม่ใช่การสอน) ที่จำเป็นต้องใช้ทั้งสองคำสั่งร่วมกัน

🧪 **แบบฝึกหัดที่ 5 (ปฏิบัติการ – สำหรับผู้ที่มีพื้นฐานเล็กน้อย):**  
เปิด Visual Studio หรือ Visual Studio Code (หรือแค่ Terminal + text editor) สร้างโปรเจกต์ Console App ชื่อ `MyFirstProject` รันโปรแกรม Hello World จากนั้นลองใช้ `Console.ForegroundColor` และ `Console.BackgroundColor` เปลี่ยนสีข้อความและพื้นหลัง (เช่น ข้อความสีเหลืองบนพื้นสีน้ำเงิน) ค้นหาวิธีการจาก IntelliSense หรือ Google แล้วบันทึกโค้ดที่ได้

## สรุปท้ายบท

บทที่ 1 นี้ได้ปูพื้นฐานให้คุณเข้าใจว่า:

- หนังสือเล่มนี้เหมาะสำหรับผู้เริ่มต้นและนักพัฒนาที่ต้องการเป็นมืออาชีพด้าน .NET
- C# เป็นภาษาเชิงวัตถุที่ทันสมัย ทำงานบนแพลตฟอร์ม .NET ซึ่งข้ามระบบปฏิบัติการได้
- หนังสือแบ่งเป็น 5 ภาค 120 บท ครอบคลุมตั้งแต่พื้นฐานภาษาไปจนถึง Redis, RabbitMQ, และหัวข้อขั้นสูง
- วิธีอ่านหนังสือให้ได้ผลคือ: อ่านตามลำดับ, ลงมือเขียนโค้ด, ทำแบบฝึกหัด, ใช้แหล่งอ้างอิง, สร้างโปรเจกต์สะสม, และเข้าร่วมชุมชน

- คุณได้ลองเขียนและรันโปรแกรม Hello World สองรูปแบบแล้ว
